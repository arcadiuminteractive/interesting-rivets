## Trading Economy Design

### Trading Tax System

**Why Taxes Matter:**
1. Creates natural gold sink (players need to buy gold)
2. Prevents inflation in player-driven economy
3. Reduces spam trading
4. Generates revenue through currency sales

**Tax Rates by Rarity:**
```lua
local TRADING_TAX = {
    Common = 100,      -- ~$0.04 in gold value
    Uncommon = 250,    -- ~$0.10
    Rare = 500,        -- ~$0.20
    Epic = 1000,       -- ~$0.40
    Legendary = 2500,  -- ~$1.00
}
```

**Tax Splits:**
- Both players pay the tax (total 2x tax per trade)
- 100% of tax is consumed (not transferred)
- Encourages purchasing gold for active traders

### Trading Frequency Limits

**Free Players:**
- 3 trades per day
- 2 items per trade
- Can only trade Common/Uncommon/Rare
- Full trading tax

**Elite Trader Pass Holders:**
- Unlimited trades per day
- 10 items per trade
- Can trade all rarities
- 75% tax reduction (pay only 25%)

### Anti-Exploit Measures

**Trade Value Validation:**
```lua
-- Both sides must have roughly equal value
local function validateTradeValue(player1Items, player2Items)
    local value1 = calculateTotalValue(player1Items)
    local value2 = calculateTotalValue(player2Items)
    
    -- Allow up to 20% value difference
    local ratio = math.max(value1, value2) / math.min(value1, value2)
    return ratio <= 1.2
end
```

**Cooldown Systems:**
- New accounts: 7-day wait before trading
- Item acquisition: 24-hour hold before tradeable
- Large-value trades (>10,000 gold equivalent): 72-hour confirmation

**Trade Logging:**
- All trades logged to DataStore
- Suspicious patterns flagged for review
- Rollback capability for exploited trades
